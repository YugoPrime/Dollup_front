## Goal
When a customer redeems loyalty points at checkout, cart total must drop by the redemption amount before order creation. Currently `redeem-apply` decrements the points ledger but does NOT inject discount into cart total — customers get charged full price. Fix this.

## Steps
1. `npx ctx7@latest library "Medusa" "completeCartWorkflow hooks add discount adjustment line"`
2. Verify existing loyalty layout: `ls -la src/modules/loyalty src/admin/widgets`
3. Create `loyalty_settings` model + migration. Single row. Fields: `earn_rate_per_100_mur` (default 1), `redeem_rate_mur_per_100_pts` (default 50), `min_redeem_points` (default 500), `welcome_bonus_points` (default 100), `points_expiry_months` (nullable)
4. Extend loyalty service: `getSettings()`, `updateSettings(input)`. Seed defaults on first call.
5. Build `workflows/apply-loyalty-discount.ts`:
   - Hook into `completeCartWorkflow` via `.hooks.beforeCartCompleted` (or v2 equivalent)
   - Read `cart.metadata.loyalty_redeem`
   - Inject fixed-amount line-item adjustment OR custom promotion (use whichever Medusa v2 sanctions)
   - Validate `discount_mur ≤ subtotal_mur`
6. Update `redeem-apply` route: confirm it writes `cart.metadata.loyalty_redeem.discount_mur` AND points-to-burn
7. Update earn logic in `subscribers/order-placed.ts` to use `settings.earn_rate_per_100_mur` (replace hardcoded)
8. Update `redeem-preview` and `redeem-apply` to use `settings.redeem_rate_mur_per_100_pts`
9. Add 5 fields to admin widget — read/write via service
10. Integration test: 500 pts redeemed → Rs 5000 cart → completeCart → order at Rs 4750
11. `yarn typecheck && yarn build` clean
12. Run migration

## Acceptance
- Customer with `≥ min_redeem_points` can redeem at checkout
- Order total = subtotal − redemption MUR equivalent
- Points ledger reflects burn correctly
- Admin widget edits all 5 knobs; changes effective immediately
- Earn rate sourced from settings (not hardcoded)
- All existing loyalty integration tests still pass
