# Phase 4.7 — blurDataURL Placeholders

**Date:** 2026-05-05
**Lane:** Frontend (`DUB-front/`)
**Priority:** P1
**Estimate:** ~45 min
**Depends on:** None
**Parallel-safe with:** Phase 6.5, Phase 6 (different repo). MUST coordinate with Phase 5 within DUB-front — see file-ownership boundary below.

## Goal
Eliminate the white flash on hero tiles and PDP main image while `next/image` loads. Use static precomputed base64 blurs (8x8) — zero runtime cost.

## Files this phase OWNS
- `scripts/generate-blur.ts` (NEW)
- `src/lib/blur-data.ts` (NEW)
- Hero component(s) — Image edits only (NOT metadata/JSON-LD)
- PDP main image component — Image edits only

## Files this phase MUST NOT TOUCH (Phase 5 territory)
- `src/app/layout.tsx` metadata block + JSON-LD
- `src/app/products/[handle]/page.tsx` `generateMetadata` export — only edit JSX `<Image>` usage
- `src/app/sitemap.ts`
- Any `metadata` export

## Steps
1. `npm i -D sharp` (skip if already present)
2. Identify Hero asset paths: `grep -rn "src=" src/components/Hero*.tsx`
3. Write `scripts/generate-blur.ts`:
   - Reads asset paths from `public/` or remote URLs
   - `sharp(input).resize(8, 8).blur().jpeg({ quality: 50 }).toBuffer()`
   - Outputs `data:image/jpeg;base64,...` per asset
4. Run script for: 4–6 hero tiles + PDP fallback placeholder
5. Emit constants to `src/lib/blur-data.ts`:
   ```ts
   export const HERO_TILE_1_BLUR = "data:image/jpeg;base64,...";
   export const PDP_FALLBACK_BLUR = "data:image/jpeg;base64,...";
   ```
6. Update Hero `<Image>` components: `placeholder="blur" blurDataURL={HERO_TILE_1_BLUR}`
7. Update PDP main `<Image>`: `placeholder="blur" blurDataURL={PDP_FALLBACK_BLUR}` (static fallback for products without their own blur)
8. `npm run build` clean
9. Visual check in dev: hard refresh hero → soft blur → sharp image, no white flash

## Acceptance
- No white flash on hero tile load
- No white flash on PDP main image load
- Build passes
- Each blur constant <300 chars
- `src/lib/blur-data.ts` checked in

## Out of scope
- Runtime blur generation for arbitrary remote Medusa product images (Phase 10 — runtime cache)
- Category / collection / wishlist card blurs
